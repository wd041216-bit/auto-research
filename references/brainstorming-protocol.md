# Brainstorming Protocol

## Purpose

Turn a rough research direction into deliberate options before committing to a proposal, contribution plan, or experiment path.

## Required Inputs

- User research direction or idea
- Existing workspace artifacts, if any
- Known constraints, assets, target venues, and risk tolerance

## Required Outputs

- Clear assumptions
- One material clarification question when needed
- Two or three candidate routes with trade-offs
- Approval or explicit provisional status before converting a route into a final proposal artifact

## Procedure

1. Inspect available artifacts before asking questions.
2. Ask only one clarification when a missing answer would materially change direction.
3. If missing details do not change the direction, state reasonable assumptions and continue.
4. Produce two or three candidate routes. Each route must include thesis, evidence path, main risk, and why it might lose.
5. Recommend one route and explain the trade-off.
6. Before finalizing a proposal or implementation path, confirm the route or label the output provisional.
7. Self-review for blank markers, contradictions, scope drift, and ambiguous requirements.

## Process Constraints

- Brainstorming may produce options, not final claims.
- Each route must include an evidence path and a failure mode.
- Original or hybrid routes that survive brainstorming must still pass the Council Gate when it applies.
- If the user asks for immediate finalization, provide a provisional route recommendation and name the missing gate.

## Blocking Gates

- Do not collapse to one route before considering alternatives unless the user supplied a fully constrained direction.
- Do not treat an unapproved route as a final proposal.
- Do not ask a stack of questions when one material question is enough.

## Common Failure Modes

- Producing a polished proposal before understanding constraints.
- Treating user excitement as evidence of feasibility.
- Asking many questions that delay useful progress.
- Skipping route comparison because one idea feels obvious.
